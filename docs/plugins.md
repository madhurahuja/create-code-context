# Provider Plugin Guide

Use plugins to add new LLM providers without changing core CLI code.

## Discovery

The runtime discovers plugins from:

- `./llm-context-plugins/*.js|*.mjs|*.cjs`
- `node_modules/create-llm-context-provider-*`
- `node_modules/@create-llm-context/provider-*`

## Contract

Implement the `ProviderPlugin` interface.

```ts
import type { ProviderPlugin } from '../src/types/plugin.js';

const plugin: ProviderPlugin = {
  name: 'my-provider-plugin',
  version: '1.0.0',
  providers: [
    {
      id: 'acme-ai',
      displayName: 'Acme AI',
      description: 'Acme provider support',
      capabilities: {
        workspaceAutomation: true,
        mcp: true,
        commands: true,
        skills: true,
        agents: true,
      },
      resolveArtifacts: ({ projectName }) => [
        {
          path: 'CONTEXT_PROMPT.md',
          content: `# ${projectName}\n\nGenerated by Acme AI provider plugin.`,
        },
      ],
    },
  ],
};

export default plugin;
```

## Artifact rules

- `path` is relative to target project root.
- Parent directories are created automatically.
- Paths should be normalized and portable.
- Return deterministic outputs to keep scaffolding reproducible.
